# ColorFit 데모 시나리오 (2분 영상용)

## 시나리오 구성: 2개 페르소나 연속 플로우

총 러닝타임: **~2분** (시나리오 A 60초 + 시나리오 D 60초)

## 시나리오 A: 진단러 김지은 (25세, Summer Cool Soft)

> **상황:** 퍼스널컬러 진단은 받았지만, 실제 쇼핑에 연결하지 못하는 사용자.
> 내일 소개팅인데, "Summer Cool Soft에 맞는 코디가 뭐지?"

| 시간 | 화면 | 액션 | 핵심 포인트 |
|------|------|------|------------|

| 0:12 | 온보딩 Step 2 | TPO: **데이트** 선택 / 스타일: **로맨틱** 선택 | 상황+분위기 설정 |

## 시나리오 D: 결정 피로형 최수빈 (28세, Summer Cool Mute)

> **상황:** 12개 코디를 저장했지만 고를 수가 없는 사용자.
> "다 괜찮아 보이는데, 결국 뭘 사야 하지?"

### 플로우 (60초)

| 시간 | 화면 | 액션 | 핵심 포인트 |
|------|------|------|------------|
| 0:00 | 피드 | (이미 로그인된 상태) 코디 피드 스크롤 | 기존 사용자 재방문 |
| 0:05 | 피드 | 하트 버튼으로 코디 2~3개 추가 저장 | 저장 기능 |
| 0:10 | 저장 목록 | 하단 탭 "저장" 이동 → 저장된 코디 목록 | 저장 목록 진입 |
| 0:15 | 저장 목록 | **TOP PICK 배너** 자동 표시: "가장 잘 어울리는 코디" | **핵심 기능: Top Pick** |
| 0:20 | Top Pick 상세 | 배너 탭 → 코디 상세 이동 → 최고점 코디 확인 | "12개 중 이게 최적" |
| 0:25 | Top Pick 상세 | 추천 이유: "Rose Gray 니트가 Summer Cool Mute의 세련된 매력을 살려요" | 명확한 근거 |
| 0:30 | 저장 목록 | 뒤로 → "A vs B" 버튼 탭 → 비교 모드 진입 | **핵심 기능: A vs B** |
| 0:35 | 저장 목록 | 코디 A 선택 (A 배지) → 코디 B 선택 (B 배지) → "비교하기" | 2개 선택 UX |
| 0:40 | 비교 페이지 | 승자 배너: "A 코디 승리!" + 판정 이유 | 즉시 결론 제시 |
| 0:45 | 비교 페이지 | 4축 나란히 비교 (PCF: A 92 vs B 78 / OF: A 88 vs B 91) | 축별 시각적 비교 |
| 0:50 | 비교 페이지 | "A가 퍼스널컬러 적합도에서 14점 더 높아요" | **가장 큰 차이 축 강조** |
| 0:55 | 코디 상세 | A 코디 탭 → 외부 구매 링크로 이동 | 결정 → 구매 |

**D 시나리오 핵심 메시지:**
> "선택 장애 해소 — Top Pick으로 1개 추천, A vs B로 비교 판정, 즉시 결정"

## 발표 시 강조할 핵심 수치

- **12-tone** 퍼스널컬러 분류 체계
- **4축 스코어링** (컬러 35% + TPO 25% + 조화 20% + 가격 20%)
- **30초** 온보딩 완료
- Top Pick **CTR 30~40%** 기대 (vs 일반 추천 15~20%)
- 최저가 비교로 **외부 구매 전환**
