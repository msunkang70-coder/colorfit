# ColorFit 사용자 검증 테스트 가이드

## 테스트 개요

| 항목 | 내용 |
|------|------|
| 기간 | 1주 |
| 대상 | 20명 (진단러 10명 + 효율러 10명) |
| 방식 | 실제 사용 + CSV 분석 + 정성 피드백 |
| URL | https://frontend-msunkang70-1055s-projects.vercel.app |

## 참여자 모집 기준

### 그룹 A: 진단러 (10명)
- 퍼스널컬러 진단 받은 경험 있음
- 진단 결과를 쇼핑에 활용하고 싶지만 방법을 모름
- 20~30대 여성

### 그룹 B: 효율러 (10명)
- 코디 고민에 시간 쓰기 싫은 직장인
- 빠르게 결정하되 근거 있는 추천을 원함
- 25~35세 남녀

## 테스트 안내 (참여자에게 전달)

```
안녕하세요! ColorFit 사용자 테스트에 참여해주셔서 감사합니다.

1. 아래 링크에 접속해주세요:
   https://frontend-msunkang70-1055s-projects.vercel.app

2. 온보딩 5단계를 진행해주세요 (성별 → 퍼스널컬러 → TPO → 예산 → 취향)

3. 피드에서 코디를 확인하고 "이 코디로 결정" 버튼을 눌러주세요

4. 간단한 설문에 답해주세요 (1~5점 신뢰도 + Yes/No 확신)

5. 원하시면 "비슷한 선택 보기"로 다른 코디도 비교해보세요

6. TPO(상황) 탭을 바꿔서 다른 상황의 코디도 시도해보세요

7. 테스트 기간 동안 자유롭게 여러 번 사용해주세요

사용 후 아래 피드백을 보내주세요:
- 코디 추천이 납득이 되었나요? (1~5)
- 가장 좋았던 점은?
- 가장 불편했던 점은?
- 실제로 이 코디를 구매할 의향이 있나요?
```

## 데이터 수집

### 정량 데이터 (자동 수집)
- TTD (결정 시간)
- CTR (결정 클릭율)
- trust_score (신뢰도 1~5)
- confidence (실행 확신 yes/no)
- expanded (Explore 진입 여부)
- selected_rank (선택 순위)

### 정성 데이터 (수동 수집)
- 사용 후 피드백 (위 질문 4개)
- 자유 의견

## 분석 방법

```bash
# 테스트 종료 후 실행
python scripts/analyze_kpi.py backend/data/metrics.jsonl
```

## 가설 판정 기준

| 가설 | 검증 데이터 | 성공 | 실패 시 대응 |
|------|-----------|------|-------------|
| H1 | TTD (Decision) 중앙값 | < 15초 | evidence 간결화, CTA 강조 |
| H2 | trust_score 평균 | >= 4.0 | risk_guard 다양화 |
| H3 | Explore trust > Decision trust | 차이 유의 | compact 카드 정보량 조절 |
| H4 | selected_rank=1 비율 | >= 60% | 스코어링 가중치 조정 |
| H5 | confidence=yes 비율 | >= 70% | risk_guard 패턴 개선 |
