## Slide 2 — 문제 정의

### 퍼스널컬러, 진단은 받았는데...

- 진단 결과를 **실제 쇼핑에 연결하지 못함** (A 진단러)

> 2030 여성 82%가 퍼스널컬러에 관심이 있지만,
> 실제 구매까지 연결하는 서비스는 없다.

**ColorFit이 해결합니다.**

## Slide 3 — 솔루션 개요

### 3가지 핵심 가치

| | 가치 | 설명 |
|---|---|---|
| 1 | **초개인화 추천** | 12-tone 퍼스널컬러 + TPO 8종 + 예산 = 나만의 코디 피드 |
| 2 | **설명 가능한 큐레이션** | "왜 이 조합이 나에게 맞는지" 4축 스코어 + 추천 이유 |
| 3 | **스마트 가격 비교** | 동일 상품 최저가 + 유사 대안 상품 제안 |

## Slide 4 — 핵심 기술: 4축 스코어링 엔진

### 추천의 과학: Weighted 4-Axis Scoring

```
Total = PCF(0.35) + OF(0.25) + CH(0.20) + PE(0.20)
```

| 축 | 이름 | 가중치 | 측정 |
|---|---|---|---|
| PCF | 퍼스널컬러 적합도 | **35%** | 12x12 톤 호환 매트릭스 |
| OF | TPO 적합도 | 25% | 아이템 태그 vs 사용자 TPO |
| CH | 색상 조화도 | 20% | 60-30-10 룰 기반 |
| PE | 가격 효율성 | 20% | 예산 대비 가격 비율 |

**왜 이 가중치인가?**
- 퍼스널컬러가 핵심 차별점 → PCF 최고 가중치
- 상황 적합성이 구매 전환에 직결 → OF 2위
- 조화도와 가격은 보조 판단 축

## Slide 5 — 페르소나 & 사용자 여정

### 4가지 페르소나

| 페르소나 | 핵심 니즈 | ColorFit 솔루션 |
|----------|----------|-----------------|
| **A 진단러** (25세) | 진단 → 구매 연결 | 추천 이유 + 톤 설명 |
| B 바쁜이 (32세) | 빠른 확신 | TPO 탭 전환 + 1줄 요약 |
| C 탐색러 (22세) | 가성비 최적화 | 유사 상품 + 예산 필터 |
| **D 결정 피로형** (28세) | 1개 강력 추천 | Top Pick + A vs B 비교 |

### 사용자 여정 (6단계)

```
인지 → 가입 → 온보딩(30초) → 탐색(피드) → 결정(Top Pick/비교) → 구매(외부 링크)
```

## Slide 6 — 데모: 시나리오 A (진단러)

### "소개팅 전날, Summer Cool Soft 코디 찾기"

```
온보딩 (12-tone 선택 → TPO → 예산)
    ↓  30초 완료
맞춤 피드 (데이트 탭 → 라벤더 블라우스 + 아이보리 슬랙스 / 89점)
    ↓
코디 상세 (4축 스코어: PCF 94 / OF 88 / CH 91 / PE 82)
  + 추천 이유: "Summer Cool Soft 시그니처 라벤더 톤이 피부를 화사하게"
    ↓
아이템 상세 (무신사 32,900원 vs W컨셉 34,500원)
  + 유사 상품 (87% 유사도)
    ↓
"무신사에서 구매하기" → 외부 쇼핑몰 이동
```

> **핵심:** 진단 결과 → 추천 이유 → 최저가 비교 → 구매까지 원스톱

## Slide 7 — 데모: 시나리오 D (결정 피로형)

### "12개 저장했는데, 결국 뭘 사야 하지?"

```
저장 목록 (12개 코디)
    ↓
TOP PICK 배너: "가장 잘 어울리는 코디"
  → Rose Gray 니트 / 총점 92점
  → "Summer Cool Mute의 세련된 매력을 살려요"
    ↓
A vs B 비교 모드 (2개 선택)
  → 축별 비교: PCF A:92 vs B:78 / OF A:88 vs B:91
  → 판정: "A 코디 승리! 퍼스널컬러 적합도에서 14점 높아요"
    ↓
결정 → 외부 구매 링크
```

> **핵심:** Top Pick으로 1개 추천, A vs B로 근거 있는 비교 → 즉시 결정

## Slide 8 — 기술 아키텍처 & 성과

### 시스템 구성

```
Next.js 15 + React 19 + TailwindCSS 4
         ↕  REST API (JSON)
Python 3.13 + FastAPI 0.115
         ↕
Recommendation Engine
  ColorMatcher → OutfitScorer → Reranker → ReasonGenerator
         ↕
PostgreSQL 17 (Supabase) + Redis 7.4 (Upstash)
  + Naver Shopping API + 12-tone Palette JSON
```

### 주요 수치

| 항목 | 수치 |
|------|------|
| 개발 기간 | 5주 (4주 개발 + 1주 발표) |
| 전체 태스크 | 67개 완료 |
| 12-tone 팔레트 | 12종 x 컬러 데이터 |
| 4축 스코어링 | 가중 합산 0~100 |
| API 엔드포인트 | 12개 |
| 프론트엔드 페이지 | 11개 |

### 배포

- **FE:** Vercel (Next.js 자동 배포)
