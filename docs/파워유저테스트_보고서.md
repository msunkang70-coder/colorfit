# ColorFit 파워유저 테스트 보고서

**테스트 일자:** 2026-04-03
**테스트 환경:** 로컬 개발서버 (localhost:3000) + 데모 페이지 (아이폰 프레임)
**테스트 기준:** 여성 / 여름쿨라이트 / 예산 3만~30만원
**테스트 범위:** 9개 TPO 탭 × E2E 흐름 (온보딩 → 피드 → Explore → 결정 → 설문)

## 1. 파워유저 페르소나 × E2E 테스트

### 👤 페르소나 A — 김지은 (25세, 사회초년생)

**상황:** 내일 첫 출근. 퍼스널컬러 진단(여름쿨라이트)은 받았지만 어떤 옷을 사야 할지 모른다.
**기대:** "내 톤에 맞는 출근 코디를 빠르게 골라주고, 왜 이건지 설명해주면 좋겠다"

#### 테스트 흐름: 출근탭

| Step | 행동 | 결과 | 판정 |
|------|------|------|------|
| 1 | 온보딩 완료 후 /feed 진입 | "오늘의 결정" + 코디 1개 표시 | ✅ |
| 2 | "출근" 탭 클릭 | 코디 변경 — "프로페셔널한 출근 조합" | ✅ |
| 3 | 코디 이미지 확인 | 니트 + 슬랙스 이미지, ₩91,700 | ✅ |
| 4 | 코디 구성 확인 | "코디 구성 2pcs" — 니트, 슬랙스 썸네일 | ✅ |
| 5 | evidence 확인 | "깔끔한 실루엣이 전문적인 분위기를 연출해요" | ✅ |
| 6 | risk_guard 확인 (CTA 위) | "✓ 니트와 슬랙스는 색상 대비가 적절해서..." | ✅ |
| 7 | "비슷한 선택 보기" 클릭 | Explore Mode — compact 카드 2개 표시 | ✅ |
| 8 | compact 카드 탭 | Decision Mode 복귀, 선택한 코디로 교체 | ✅ |
| 9 | "이 코디로 결정" 클릭 | 설문 바텀시트 팝업 | ✅ |
| 10 | 설문 응답 (신뢰도 4, 확신 Yes) | 쇼핑몰 이동 + 로그 저장 | ✅ |

**혜택:**
- 출근에 뭘 입어야 할지 고민하는 시간이 **20분 → 10초로 단축**
- "왜 이 코디인지" 설명이 있어서 확신을 갖고 구매 가능
- 다른 선택지도 확인할 수 있어서 "더 좋은 게 있지 않을까" 불안 해소

**불편한 점:**
- 코디 이미지가 니트 단품 사진이라 "코디 전체"가 상상이 안 됨
- 출근 코디가 10건뿐이라 선택지 부족
- evidence가 1줄이라 설득력이 약할 수 있음

### 👤 페르소나 B — 박민준 (32세, 직장 3년차)

**상황:** 다음 주 경력직 면접. 첫인상이 중요한데 뭘 입어야 할지 모른다.
**기대:** "면접에 딱 맞는 정장 코디를 빠르게 찾고, 실패하지 않을 코디를 원한다"

#### 테스트 흐름: 면접탭

| Step | 행동 | 결과 | 판정 |
|------|------|------|------|
| 1 | "면접" 탭 클릭 | "면접관에게 좋은 인상을 주는 코디" | ✅ |
| 2 | 코디 확인 | 셔츠 + 슬랙스 + 힐, ₩174,870 | ✅ |
| 3 | evidence | "단정한 핏과 차분한 색감으로 성실한 인상을 줘요" | ✅ |
| 4 | risk_guard | "색상 대비가 적절해서 튀거나 칙칙해 보일 가능성이 낮아요" | ✅ |
| 5 | Explore Mode | compact 카드 표시 | ✅ |
| 6 | 결정 → 설문 → 이동 | 정상 동작 | ✅ |

**혜택:**
- "면접관에게 좋은 인상"이라는 문구가 **면접이라는 맥락에 딱 맞는 안심 요소**
- risk_guard가 "실패하지 않는다"는 확신을 줌
- 3건 중 최적 1개를 추천하므로 선택 과부하 없음

**불편한 점:**
- 면접 코디가 3건뿐 — 다양성 부족
- 이미지가 정장 세트가 아니라 개별 아이템이라 "면접 정장" 느낌 약함
- 남성 기준으로 온보딩하면 결과 수가 더 적을 수 있음

### 👤 페르소나 C — 이수진 (28세, 프리랜서)

**상황:** 주말 데이트. 자연스럽게 예쁘게 보이고 싶다. 너무 과하지 않은 코디.
**기대:** "데이트에 맞는 분위기 코디, 과하지 않으면서 센스 있는 것"

#### 테스트 흐름: 데이트탭

| Step | 행동 | 결과 | 판정 |
|------|------|------|------|
| 1 | "데이트" 탭 클릭 | "데이트에서 호감을 주는 코디" | ✅ |
| 2 | 코디 확인 | 원피스 + 슬랙스 + 힐, ₩207,420 | ⚠️ |
| 3 | evidence | "자연스러운 스타일링으로 부담 없이 호감을 줘요" | ✅ |
| 4 | Explore | 9건 중 Top3 비교 가능 | ✅ |
| 5 | 결정 | 정상 | ✅ |

**혜택:**
- "부담 없이 호감"이라는 표현이 데이트 맥락과 잘 맞음
- Explore Mode에서 가격대/스타일 다른 코디 비교 가능

**불편한 점:**
- ₩207,420은 데이트 코디치고 고가 — 예산 필터 조정 필요
- "원피스 + 슬랙스" 조합이 어색함 (원피스에 슬랙스를 함께?)
- 이미지가 원피스 여러 벌 진열 사진이라 코디 느낌 아님

### 👤 페르소나 D — 최영호 (23세, 대학생)

**상황:** 학교 갈 때 편하면서 멋진 코디. 가성비 중요.
**기대:** "편한데 멋있는, 너무 비싸지 않은 코디"

| Step | 행동 | 결과 | 판정 |
|------|------|------|------|

## 4. 유사 서비스 대비 차별점 3가지

### 차별점 1: "결정 구조" vs "탐색 구조"

| | 무신사/W컨셉 | ColorFit |
|---|---|---|
| 제공 방식 | 상품 수백 개 나열 → 사용자가 고름 | 코디 1개 제시 → 사용자가 확인 |
| 사용자 역할 | 탐색자 (스크롤 + 비교) | 결정자 (확인 + 수락) |
| 소요 시간 | 20~40분 | 10~15초 |

**"추천이 아니라 결정을 만든다"** — 선택지를 줄여서 결정 속도를 높인다.

### 차별점 2: "이유 기반 설득" vs "감각 기반 탐색"

| | 기존 서비스 | ColorFit |
|---|---|---|
| 추천 근거 | 없음 ("당신을 위한 추천") | core + evidence + risk_guard 3파트 |
| 사용자 반응 | "왜 이건데?" → 무시 | "아, 그래서 이거구나" → 납득 |
| 신뢰 구조 | 알고리즘 블랙박스 | 이유가 명시적으로 보임 |

**"왜 이 코디인지" + "왜 실패하지 않는지"를 말해주는 유일한 서비스**

### 차별점 3: "퍼스널컬러 × TPO × 예산" 교차 필터

| | 잼페이스/Dressika | Stitch Fix | ColorFit |
|---|---|---|---|
| 퍼스널컬러 | 진단만 | 없음 | 진단 + 코디 연결 |
| TPO (상황) | 없음 | 스타일리스트 주관 | 8개 TPO 자동 필터 |
| 예산 | 없음 | 고정 가격대 | 슬라이더로 실시간 조절 |

**퍼스널컬러 진단 결과를 "실제 쇼핑에 연결"하는 유일한 서비스**

## 6. 서비스 성숙도 평가

| 항목 | 현재 수준 | 목표 수준 | 갭 |
|------|----------|----------|-----|
| 결정 구조 (Decision Mode) | 80% | 95% | evidence 강화, CTA 설득력 |
| 비교 구조 (Explore Mode) | 70% | 90% | compact 카드 차별화, 선택 이유 명확화 |
| 데이터 품질 | 50% | 85% | 운동복 데이터, 코디 이미지, 조합 다양성 |
| 설득 텍스트 | 75% | 90% | TPO 특화 완료, risk_guard 다양화 필요 |
| 측정 체계 | 90% | 95% | scroll depth 추가 |
| 디자인 완성도 | 65% | 85% | 매거진 감도, 여백, 타이포 |

### 한 줄 총평

> **"결정 구조와 설득 텍스트는 잘 잡혔다. 이제 데이터 품질(코디 이미지 + 운동복 + 조합 다양성)이 서비스 품질의 병목이다."**
