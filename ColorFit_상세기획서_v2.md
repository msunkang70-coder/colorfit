## 1. 서비스 재정의

### 1.1 v1 → v2 전환

| | v1 (추천 서비스) | v2 (의사결정 지원 서비스) |
|---|---|---|
| **정체성** | 코디를 추천한다 | 결정을 쉽게 만든다 |
| **출력** | 코디 리스트 (N개) | 결정 1개 + 선택적 비교 (3~5개) |
| **이유** | "어울려요" (결론) | core + evidence + risk_guard (근거) |
| **사용자 역할** | 탐색자 (스크롤 → 선택) | 결정자 (확인 → 수락 또는 비교 → 선택) |
| **성공 기준** | 세션 시간, 저장 수 | TTD, CTR, 신뢰도, 실행 확신 |

### 1.2 서비스 정의

> **ColorFit은 퍼스널컬러, 상황, 예산을 분석해서 최적의 코디 하나를 제안하고, 왜 이 코디인지 설명하고, 원하면 비교 선택지를 제공해서 사용자가 빠르고 확신 있게 결정하도록 만드는 서비스다.**

핵심 키워드: **Guided Decision** — 안내된 결정

### 1.3 핵심 원칙

1. **Decision First.** 기본은 코디 1개. 리스트를 보여주지 않는다.
2. **Explain Why.** 모든 제안에는 근거가 있다. "왜 이건지"와 "왜 괜찮은지"를 말한다.
3. **Compare When Needed.** 비교는 사용자가 원할 때만. 최대 5개, 각각 다른 이유를 가진다.
4. **Measure Decisions.** 추천 품질이 아니라 결정 속도와 확신을 측정한다.

### 1.4 Vision / Mission

**Vision:**
> "옷 고르는 시간을 0으로 만든다" — 누구나 퍼스널컬러 전문가처럼 코디를 결정하는 세상

**Mission:**
> 퍼스널컬러 진단을 받았지만 쇼핑에 활용하지 못하는 사람들에게(Who), 상황·예산·취향에 맞는 최적 코디와 그 근거를(What), 5축 데이터 분석 + 심리 설계 기반 의사결정 지원으로(How) 제공한다.

**Core Values:**

| 가치 | 정의 | 행동 지표 |
|------|------|----------|
| 결정 우선 (Decision First) | 탐색이 아닌 결정을 설계한다 | 모든 화면은 "다음 액션"이 명확하다 |
| 설명 가능 (Explainable) | 추천에는 반드시 근거가 있다 | evidence 없는 추천은 출력하지 않는다 |
| 측정 기반 (Data-Driven) | 감이 아닌 데이터로 판단한다 | 모든 UX 변경은 TTD/CTR 영향을 측정한다 |

## 2. 문제 정의 및 시장 분석

### 2.1 세 겹의 결정 장벽

| 층 | 장벽 | 사용자 심리 | ColorFit 대응 |
|---|---|---|---|
| **1층** | 선택 과부하 | "뭘 골라야 하지" | Decision Mode: Top1만 제시 |
| **2층** | 신뢰 부족 | "왜 이건데?" | evidence: 축 기반 근거 문장 |
| **3층** | 리스크 불안 | "이거 입고 나갔다가 망하면?" | risk_guard: 실패 가능성이 낮은 구조적 이유 |
| **3.5층** | 비교 결핍 | "다른 건 없었을까?" | Explore Mode: Top3~5 비교 후 확신 |

### 2.2 기존 시장의 한계

### 2.4 시장 규모 분석 (TAM/SAM/SOM)

**산출 방법:** Top-Down + Bottom-Up 교차 검증

#### Top-Down 접근

| 구분 | 규모 | 산출 근거 |
|------|------|----------|
| **TAM** | 약 30조원 | 한국 온라인 패션 시장 (2025, 통계청 기준 연 8% 성장) |
| **SAM** | 약 3,000억원 | TAM × 퍼스널컬러 관심 비율(~10%) × 코디 추천 니즈 비율(~10%) |
| **SOM** | 약 15억원 | SAM × 목표 점유율 0.5% (서비스 론칭 1년차) |

#### Bottom-Up 접근

| 항목 | 수치 | 근거 |
|------|------|------|
| 잠재 사용자 풀 | 약 500만명 | 20~30대 여성 중 퍼스널컬러 진단 경험자 (전체 2,000만 × 25%) |
| 서비스 전환율 | 2% | 유사 서비스 초기 전환율 벤치마크 |
| 예상 활성 사용자 | 100,000명 | 500만 × 2% |
| ARPU (연간) | 15,000원 | CPS 수수료 기반 (평균 구매 10만원 × 클릭당 전환 5% × 수수료 3%) |
| **SOM (Bottom-Up)** | **약 15억원** | 100,000 × 15,000원 |

**교차 검증:** Top-Down SOM(15억) ≈ Bottom-Up SOM(15억) → 일관성 확인

**시장 성장 동인:**
- 퍼스널컬러 진단 시장 연 20%+ 성장 (잼페이스/톤코 등 진단 앱 급증)
- MZ세대 개인화 소비 트렌드 강화
- AI 기반 패션 추천 시장 CAGR 15%+ (글로벌)
- 패션 이커머스 지속 성장 (오프라인 → 온라인 전환)

### 2.5 산업 구조 분석 (Porter's Five Forces)

| 요인 | 점수 (1-5) | 수준 | 핵심 근거 |
|------|-----------|------|----------|
| **기존 경쟁자 간 경쟁** | 3 | 중간 | 패션 플랫폼 다수, 그러나 "퍼스널컬러 기반 결정 지원"은 공백 |
| **신규 진입자 위협** | 4 | 높음 | 기술 장벽 낮음 (API 기반), 대형 플랫폼의 기능 추가 가능 |
| **대체재 위협** | 4 | 높음 | 인간 스타일리스트, SNS 참고, 자체 검색, "그냥 안 사기" |
| **공급자 교섭력** | 2 | 낮음 | 네이버 쇼핑 API 공개, 상품 데이터 풍부, 다중 데이터 소스 |
| **구매자 교섭력** | 4 | 높음 | 무료 대안 다수, 전환 비용 0, 가격 민감도 높음 |
| **산업 매력도** | **2.6** | **보통** | (5 - 3.4) / 4 × 100 = 40% |

**전략적 시사점:**
- 신규 진입 + 구매자 교섭력 높음 → **차별화 해자(moat) 구축 필수**
- 해자 전략: (1) 퍼스널컬러×5축 스코어링 알고리즘, (2) selected_rank 학습 데이터 축적, (3) evidence/risk_guard 설명 품질
- 공급자 교섭력 낮음 → 데이터 확보 유리, 멀티 소스 전략 가능

### 4.2 BSC (Balanced Scorecard)

OKR의 KR을 4개 관점으로 매핑하여 전략적 균형을 확인한다.

```
[학습·성장] selected_rank 데이터 축적 → 축 가중치 학습
    ↓
[내부 프로세스] Feed API 800ms 이내 + 5축 스코어링 정확도 + reason 품질
    ↓
[고객] TTD < 15초 + 신뢰도 ≥ 4.0 + 확신 ≥ 70%
    ↓
[재무] CTR > 30% → CPS 전환 → ARPU 15,000원/년
```

| 관점 | KPI | 유형 | OKR 연결 | 현재 | 목표 | 측정 주기 |
|------|-----|------|----------|------|------|----------|
| **재무** | CTR (CTA 클릭율) | Lag | O3-KR2 | - | > 30% | 주간 |
| **재무** | ARPU (유저당 연매출) | Lag | - | - | 15,000원 | 월간 |
| **고객** | TTD (Decision Mode) | Lag | O1-KR1 | - | < 15초 | 주간 |
| **고객** | 신뢰도 평균 | Lag | O1-KR2 | - | ≥ 4.0 | 주간 |
| **고객** | 실행 확신 Yes% | Lag | O1-KR3 | - | ≥ 70% | 주간 |
| **고객** | 온보딩 완주율 | Lag | O3-KR1 | - | ≥ 60% | 주간 |
| **프로세스** | Feed API 응답 시간 | Lead | - | - | < 800ms | 일간 |
| **프로세스** | 추천 이유 생성 성공율 | Lead | - | - | > 95% | 일간 |
| **프로세스** | 데이터 품질 점수 | Lead | - | - | 위반 코디 0건 | 주간 |
| **학습·성장** | selected_rank 수집량 | Lead | O2-KR2 | 0 | 200건+ | 주간 |
| **학습·성장** | 사용자 정성 피드백 수집 | Lead | O3-KR3 | 0 | 20명 | 1회 |

**Lead:Lag 비율 = 5:6 ≈ 45:55** (목표 60:40에 비해 Lag 비중 높음 → MVP 특성상 허용, 이후 Lead 지표 보강)

## 6. 추천 시스템 재정의

### 6.1 파이프라인

```
사용자 프로필 (톤, TPO, 성별, 예산)
      │
      ▼
Hard Filter (H1~H8)
      │
      ▼
Soft Score (5축 가중합)
      │
      ▼
Rerank (완성도 가산, 톤 다양성, 중복 제거)
      │
      ▼
Top-N 후보 (총점순, ~200개)
      │
      ▼ ── 여기까지 기존과 동일 ──
      │
Reason Gen (코디별 core / evidence / risk_guard)
      │
      ▼
프론트 수신 (page_size=5)
      │
      ▼
축 기반 Top3 선발 (프론트에서 실행)
      │
      ├─ Top1: 총점 1위 (종합 추천)
      ├─ Top2: Top1과 다른 1순위 축 (예: sf → 실루엣형)
      └─ Top3: Top1,2와 다른 1순위 축 (예: pe → 가성비형)
      │
      ▼
Decision Mode: Top1만 노출
Explore Mode: Top1 + Top2 + Top3 노출
```

### 6.2 5축 스코어링 (기존 유지)

| 축 | 이름 | 가중치 | 역할 |
|----|------|--------|------|
| PCF | Personal Color Fit | 0.25 | 퍼스널컬러 적합도 |
| OF | Occasion Fit | 0.20 | TPO 적합도 |
| CH | Color Harmony | 0.15 | 아이템 간 색상 조화 |
| PE | Price Efficiency | 0.15 | 예산 대비 가격 효율 |
| SF | Style Fit | 0.25 | 실루엣·포멀도·카테고리 궁합 |

### 6.3 이유 생성 구조 (core / evidence / risk_guard)

| 파트 | 역할 | 데이터 소스 | 예시 |
|------|------|------------|------|
| **core** | 코디 요약 한 줄 | 아이템 카테고리 + 톤 + TPO | "니트 + 슬랙스 — 봄웜라이트 출근룩" |
| **evidence** | 왜 이 코디인지 (설득) | 1순위 축 + 아이템 데이터 | "니트의 색감이 봄웜라이트 톤과 자연스럽게 어울려서 피부가 밝아 보여요" |
| **risk_guard** | 왜 실패하지 않는지 (안전) | 색상 조화/포멀도/TPO 범위 | "니트와 슬랙스는 색상 대비가 적절해서 튀거나 칙칙해 보일 가능성이 낮아요" |

### 6.4 Top3 차별화 전략

각 코디가 **다른 축에서 가장 강한 것**으로 선발되므로, evidence가 자동으로 다른 관점을 제공한다.

| 순위 | 선발 기준 | 라벨 | evidence 축 |
|------|----------|------|------------|
| Top1 | 총점 1위 | 종합 추천 / 1위 추천 | (코디에 따라 다름) |
| Top2 | Top1과 다른 1순위 축 보유, 그 중 총점 최고 | {축 라벨}형 | Top1과 다른 축 |
| Top3 | Top1,2와 다른 1순위 축 보유, 그 중 총점 최고 | {축 라벨}형 | Top1,2와 다른 축 |

**축 → 라벨:**
pcf=컬러 매칭형, of=상황 최적형, ch=색감 조화형, pe=가성비형, sf=실루엣형

**fallback:** 5개 코디가 모두 같은 축이면 총점 순서대로 "1위/2위/3위 추천" 라벨 사용.

## 7. 심리 설계

### 7.1 선택 피로 감소 — Decision Mode

| 원리 | 적용 |
|------|------|
| 선택지 역설 (Paradox of Choice) | 기본 1개만 제시. 리스트 없음 |
| 기본값 효과 (Default Effect) | Top1이 기본값. "아무것도 안 하면 이게 결정" |
| 인지 부하 최소화 | core 1줄 + evidence 1줄 + risk_guard 1줄. 스코어 점수 미노출 |

### 7.2 손실 회피 — risk_guard

| 원리 | 적용 |
|------|------|
| 손실 회피 (Loss Aversion) | "실패할 가능성이 낮다"는 표현으로 잠재 손실 제거 |
| 프레이밍 효과 | "이 색상은 안전하다" (긍정 프레이밍, "위험하지 않다"가 아님) |

### 7.3 후회 최소화 — Explore Mode

| 원리 | 적용 |
|------|------|
| 후회 회피 (Regret Aversion) | "다른 선택지도 확인했다"는 사실이 후회를 줄인다 |
| FOMO 해소 | Top3를 봤으므로 "놓친 게 없다"는 심리적 완결 |
| 선택 확인 편향 (Choice-supportive Bias) | 비교 후 Top1을 다시 선택하면 확신이 강화된다 |

### 7.4 비교 기반 확신 — Top3 차별화 라벨

| 원리 | 적용 |
|------|------|
| 대비 효과 (Contrast Effect) | 각 카드에 다른 라벨 → "이건 이 점이 강하고, 저건 저 점이 강하다" |
| 자기결정감 (Self-determination) | 사용자가 직접 선택 → "내가 고른 것"이라는 소유감 |

### 9.2 페르소나 A — 진단러 김지은 (25세)

| 항목 | 내용 |
|------|------|
| 퍼스널컬러 | 여름쿨소프트 |
| 직업 | 대학원생 |

| 구매 결정 요인 | 톤 적합성 > 가격 > 트렌드 |

**시나리오: 데이트 코디 (Decision Mode)**

```
1. ColorFit 실행 → TPO "데이트" 탭
2. Top1 표시: 라벤더 블라우스 + 아이보리 스커트
   core: "블라우스 + 스커트 — 여름쿨소프트 데이트룩"
   evidence: "블라우스의 색감이 여름쿨소프트 톤과 자연스럽게 어울려서 피부가 밝아 보여요"
   risk_guard: "블라우스와 스커트는 색상 대비가 적절해서 튀거나 칙칙해 보일 가능성이 낮아요"
3. "이걸로 결정" 탭 → 설문 (신뢰도 5, 확신 Yes) → 쇼핑몰 이동
   TTD: 8초
```

### 9.3 페르소나 B — 효율러 박민준 (32세)

| 항목 | 내용 |
|------|------|
| 퍼스널컬러 | 가을웜딥 |
| 직업 | IT 회사 팀장 |
| 핵심 불편 | 코디 고민할 시간 없음. 사도 후회할까봐 결정 못 함 |
| 정보 탐색 채널 | 유튜브, 무신사 |
| 구매 결정 요인 | 시간 절약 > 적합성 > 가격 |
| 대표 발언 | "빨리 정해주되, 다른 선택지도 보고 싶다" |

**시나리오: 출근 코디 (Explore Mode)**

```
1. ColorFit 실행 → TPO "출근" (기본)
2. Top1 표시: 셔츠 + 슬랙스 — 가을웜딥 출근룩
   "이게 좋긴 한데, 다른 것도 볼까?"
3. "비슷한 선택 보기" 탭
4. Top3 표시:
   [1위 추천] 셔츠 + 슬랙스 (컬러 매칭형)
   [실루엣형] 니트 + 와이드팬츠 — 실루엣 밸런스 강점
   [가성비형] 맨투맨 + 청바지 — ₩48,000 예산 최적
5. "역시 1위가 낫다" → Top1 탭 → Decision Mode 복귀
6. "이걸로 결정" → 설문 (신뢰도 4, 확신 Yes) → 쇼핑몰 이동
   TTD: 22초 (비교 포함)
```

## 12. 차별화 포인트

### 12.1 vs 추천 서비스

| 일반 추천 | ColorFit |
|-----------|---------|
| "이것도 좋고 저것도 좋아요" | "이게 가장 좋고, 왜 좋은지 이거예요" |
| 사용자가 고른다 | 서비스가 고르고, 사용자가 확인한다 |
| 세션 시간이 KPI | 결정 속도가 KPI |
| 많이 보여주면 좋다 | 적게 보여주되 확신을 준다 |

### 12.2 vs 스타일리스트

| 인간 스타일리스트 | ColorFit |
|------------------|---------|
| 비용 높음, 시간 필요 | 무료, 즉시 |
| 설명 가능 (대화) | 설명 가능 (evidence + risk_guard) |
| 비교 어려움 | 비교 가능 (Explore Mode) |
| 주관적 | 5축 데이터 기반 |

### 12.3 vs "전문가"라는 말의 근거

| 전문가 속성 | ColorFit 구현 |
|------------|--------------|
| 개인 맥락 파악 | 12톤 퍼스널컬러 + TPO + 예산 + 취향 시드 반영 |
| 논리적 설명 | evidence (축 기반 인과 문장) + risk_guard (구조적 안전 근거) |
| 다양한 관점 | Top3 축 기반 차별화 (컬러형/실루엣형/가성비형 등) |
| 학습 | selected_rank 데이터 → 선호 축 패턴 파악 (향후 가중치 조정) |

### 12.4 핵심 한 줄

**"ColorFit은 추천하지 않는다. 결정을 쉽게 만든다."**

| H2: 설명이 신뢰를 높인다 | trust_score 평균 | ≥ 4.0 | risk_guard 패턴 다양화, 톤 설명 구체화 |

### 13.4 개인 취향 학습 구조 (향후)

현재 MVP에서는 수집만 한다. 학습 적용은 향후.

```
selected_rank 데이터 축적
  → 사용자가 sf(실루엣형)를 반복 선택하면
  → sf 가중치를 0.25 → 0.30으로 상향
  → Top1이 실루엣 강점 코디로 변경
  → 더 빠른 Decision Mode 결정
```

수집 필드: `expanded`, `expand_level`, `selected_rank`
학습 시점: 10건+ 축적 시 (MVP 이후)
